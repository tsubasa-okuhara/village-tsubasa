## 検討中の利用内容

1. **対象契約書**
   - ヘルパーとの雇用契約書（新規・更新）
   - 事業所間の業務委託契約・秘密保持契約
   - 利用者向け 重要事項説明書・利用契約書（**代理人署名あり**）

2. **想定件数（年間）**
   - 雇用契約: 20〜40 件 ＋ 更新通知（全ヘルパーぶん）
   - 業務委託・秘密保持: 5〜10 件
   - 利用者契約: 20〜30 件（新規）＋ 改訂時の全利用者ぶん（約 116 名）
   - 合計: 初年度 150〜300 件程度を想定

3. **連携方式**
   - 自社アプリ（Firebase Functions / Node.js）から Web API を呼び出す構成
   - **自社 UI 完結**で、利用者・ヘルパーが直接クラウドサインの画面に触れないフロー
   - Webhook で署名完了通知を受け取り、自社 DB（Supabase）に契約状態を反映

## ご確認したい事項

1. **契約プラン・料金**
   - Web API を使う場合に最適なプラン
   - 送信単価（1契約あたり）
   - 初期費用・月額費用・API利用に伴う追加費用の有無

2. **API 仕様**
   - 最新の Web API 仕様書（SwaggerHub で公開されている内容以外に NDA 必要なものがあるか）
   - **代理人署名**（本人に代わって家族や成年後見人が署名）に対応する推奨の実装パターン
   - 署名位置（widget）の指定方法のベストプラクティス
   - Webhook の種類・ペイロード・署名検証方式

3. **介護業界での導入実績**
   - 訪問介護事業所での導入事例の共有可否
   - 重要事項説明書・利用契約書を電子契約化した事例があれば参考にしたい

4. **サンドボックス / 検証環境**
   - 本契約前に検証環境で API を叩けるか
   - 実送信料金が発生しない形での検証手段

5. **既存書面との並行運用**
   - 紙契約と電子契約を併用する場合の推奨運用（電子化できない利用者が残ることを想定）
